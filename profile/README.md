# iKnow

`iKnow`는 수업 중 학생의 이해 어려움 신호를 실시간으로 감지하고, 강의 맥락과 함께 해석하여 즉각적인 교수 개입까지 연결하는 AI 기반 교육 지원 플랫폼입니다. 대회 출품작으로서 `iKnow`는 “학생이 어려워하는 순간을 수업이 끝난 뒤가 아니라, 바로 그 순간에 발견할 수 없을까?”라는 문제의식에서 출발했습니다.

기존 수업 환경에서는 학생의 표정, 집중 이탈, 수동 질문 요청, 반복되는 이해 실패가 산발적으로 발생해도 이를 강사가 정량적으로 파악하기 어렵습니다. `iKnow`는 이 문제를 해결하기 위해 시각 신호 분석, 실시간 알림, 강의 구간 요약, 대시보드 기반 회고 분석을 하나의 서비스 흐름으로 통합했습니다. 이를 통해 수업 중에는 빠른 개입을 지원하고, 수업 후에는 데이터 기반 개선 방향을 제시합니다.

## Project Summary

`iKnow`는 학생용 인터페이스, 강사용 실시간 운영 화면, 관리자/강사용 대시보드, AI 분석 서버, 백엔드 집계 서버로 구성됩니다. 학생 단에서는 웹캠 프레임 분석과 수동 도움 요청을 통해 학습 혼란 신호를 수집하고, 강사 단에서는 해당 신호를 실시간 알림으로 수신합니다. 이후 강의 구간을 음성 기록과 함께 요약하여 “어떤 개념에서 이해가 막혔는지”를 설명 가능한 형태로 제공합니다.

대시보드에서는 반별 알림 수, 시간대별 이해 어려움 추이, 주요 키워드, 신호 유형 비중, AI 코칭 결과를 종합 제공하여 단순 모니터링을 넘어 실제 수업 개선에 활용 가능한 데이터 자산을 구축합니다.

## Key Features

- 학생 웹캠 프레임 기반 실시간 이해 어려움 신호 감지
- 수동 도움 요청 버튼을 통한 즉시 개입 요청 지원
- 세션 단위 WebSocket 실시간 알림 전달
- 강의 구간 STT 기록 및 AI 요약 생성
- 주요 키워드, 재설명 필요 개념, 최근 알림 이력 제공
- 시간대별 이해 어려움 추이 및 반별 KPI 대시보드 제공
- 대시보드 통계를 활용한 생성형 AI 코칭 메시지 추천

## Competitive Differentiation

`iKnow`의 차별점은 단순히 “학생 상태를 분류하는 AI”에 머무르지 않는다는 점입니다. 본 서비스는 감지, 저장, 요약, 시각화, 코칭 추천까지 이어지는 전주기 구조를 갖추고 있습니다. 즉, AI가 혼란 신호를 감지하는 것에서 끝나는 것이 아니라, 강사가 바로 활용할 수 있는 행동 가능한 정보로 전환하는 데 초점을 맞추고 있습니다.

또한 학생 개인의 신호를 세션 단위, 반 단위, 시간대 단위로 재구성함으로써 실시간 운영성과 사후 분석 가치를 동시에 확보했습니다. 이는 대회 관점에서 기술 구현을 넘어 실제 교육 현장 적용 가능성을 보여주는 핵심 요소입니다.

## System Composition

### `React`

React 기반 프론트엔드 애플리케이션으로, 학생 화면, 강사 화면, 대시보드를 포함합니다.  
학생 화면은 세션 참여, 프레임 업로드, 수동 요청을 담당하고, 강사 화면은 실시간 알림 수신, 세션 운영, STT 기반 강의 기록을 담당합니다. 대시보드는 반별 학습 신호를 종합적으로 시각화하여 운영자와 강사가 수업 흐름을 한눈에 파악할 수 있도록 설계되었습니다.

### `Spring Boot`

Spring Boot 기반 백엔드 서버로, 세션 관리, 참여자 관리, 학습 신호 저장, 알림 관리, 요약 데이터 저장, 커리큘럼 관리, 대시보드 집계, 이해 어려움 추이 계산을 수행합니다. WebSocket 기반 실시간 알림 전달과 MySQL 기반 데이터 축적을 통해 운영성과 분석 기능을 안정적으로 지원합니다.

### `FastAPI`

FastAPI 기반 AI 서비스로, 학생 프레임 이미지 분석, 강의 텍스트 요약, 대시보드 기반 AI 코칭 생성을 담당합니다. `FER`, `MediaPipe`, `OpenAI API`를 결합하여 표정·시선·얼굴 특징과 강의 텍스트를 함께 해석하고, 이를 설명 가능한 AI 결과로 반환합니다.

## Technical Architecture

| Domain | Stack |
| --- | --- |
| Frontend | React 19, Vite, React Router DOM, Axios, Recharts, STOMP/SockJS |
| Backend | Java 21, Spring Boot, Spring Data JPA, MySQL 8, Spring WebSocket |
| AI Service | Python 3.11, FastAPI, FER, MediaPipe, OpenAI API |
| Media / Interaction | Web Speech API, MediaRecorder API, WebSocket |

서비스는 프론트엔드, 백엔드, AI 서버를 분리한 구조로 설계되어 있어 기능 확장과 유지보수에 유리합니다. 학생 클라이언트는 AI 서버와 백엔드 서버를 병행 호출하고, 백엔드는 모든 이벤트를 집계해 실시간 운영과 사후 분석 모두에 활용합니다.

## Expected Impact

`iKnow`는 수업 중 학생의 이해 단절을 조기에 발견함으로써 강사의 대응 속도를 높이고, 수업 후에는 어떤 개념과 시간대에서 학습 저항이 집중되었는지를 정량적으로 확인할 수 있게 합니다. 이는 직관과 경험에 의존하던 교육 운영을 데이터 기반 의사결정으로 전환하는 데 기여합니다.

특히 본 프로젝트는 “실시간 교육 개입”과 “수업 품질 개선 분석”을 하나의 플랫폼 안에서 연결했다는 점에서 교육 현장 적용성과 확장 가능성이 높습니다. 향후에는 학교, 학원, 사내 교육, 온라인 라이브 클래스 등 다양한 러닝 환경으로의 확장이 가능합니다.

## Vision

`iKnow`는 학습자의 작은 어려움도 놓치지 않는 수업 환경을 만드는 것을 목표로 합니다. 대회 출품작으로서 본 프로젝트는 AI 기술을 단순 시연하는 수준을 넘어, 교육 현장에서 즉시 활용 가능한 사용자 경험과 운영 구조까지 포함한 실전형 서비스 모델을 제안합니다.
